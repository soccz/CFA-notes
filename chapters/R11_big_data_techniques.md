## LOS 11.a: 핀테크 (FinTech)

### 상황 속으로

당신이 주식을 사려고 합니다. 20년 전이라면 재무제표를 읽고 뉴스를 찾아보는 정도가 전부였습니다.

하지만 지금은 이런 일이 벌어집니다:
- 어떤 헤지펀드는 **위성 사진**으로 월마트 주차장에 차가 몇 대 있는지 세서 매출을 예측합니다
- 다른 펀드는 소셜 미디어에서 브랜드 감성을 실시간 분석합니다
- 신용카드 거래 데이터로 스타벅스의 이번 주 결제 건수를 분기 실적 전에 추정하기도 합니다

숫자를 비교해 봅시다:

- **펀드 A**: 재무제표만 분석. 실적 발표 후 주가 5% 상승 확인. 이미 늦었다.
- **펀드 B**: 위성 사진 + 신용카드 데이터 분석. 실적 발표 2주 전에 매출 증가를 예측하고 매수. 5% 수익 확보.

**왜 이런 결과가 나올까?** 같은 정보라도 **먼저 아는 사람**이 돈을 번다. 문제는 이 데이터의 양이 인간이 처리할 수 있는 범위를 넘어선다는 것이다. 컴퓨터가 대신 패턴을 찾아주는 기술 — 머신러닝과 AI — 이 필요해진 것이다.

이제 정리하면: **핀테크(FinTech)** = 금융 서비스 산업에 적용되는 기술 발전. ATM에서 알고리즘 트레이딩까지 모두 포함된다. 빅데이터와 AI는 **남들이 못 보는 정보에서 먼저 패턴을 찾아 투자 우위를 확보하는 도구**다.

#### 핀테크의 두 기둥

CFA 교재에서 강조하는 핀테크의 핵심 발전 영역:

1. **데이터 처리 능력 확대**: 다양한 소스에서 대량의 데이터를 수집·저장·처리하는 인프라
2. **AI/ML 분석 도구**: 수집한 데이터에서 의미 있는 패턴을 추출하는 알고리즘

**시나리오 — ATM에서 로보어드바이저까지:**

| 시대 | 핀테크 혁신 | 영향 |
|-----|-----------|------|
| 1960s | ATM | 은행 창구 없이 현금 인출 |
| 1990s | 온라인 뱅킹 | 물리적 지점 방문 불필요 |
| 2000s | 알고리즘 트레이딩 | 밀리초 단위 매매 |
| 2010s | 로보어드바이저 | 수수료 1/10로 자산 관리 |
| 2020s | AI 기반 신용평가 | 은행 이력 없어도 대출 가능 |

**왜 이런 진화가 일어나는가?** 각 혁신은 **비용을 줄이거나 속도를 높이거나 접근성을 넓힙니다.** ATM은 인건비를 줄였고, 온라인 뱅킹은 물리적 비용을 줄였고, AI 신용평가는 기존에 서비스받지 못하던 고객(unbanked)까지 포함시켰습니다.

**숫자로 느끼기:**
- 전통 자산관리: 연 1.5% 수수료 × $1M 포트폴리오 = $15,000/년
- 로보어드바이저: 연 0.25% 수수료 × $1M = $2,500/년
- 30년간 수수료 차이: 약 $375,000 (복리 효과 포함)

**변수를 바꾸면?** 포트폴리오가 $100K이면 전통 매니저는 수지가 안 맞아 서비스를 거부합니다. 로보어드바이저는 $250/년만 받고 서비스합니다. 기술이 **금융 민주화**를 이끈 것입니다.

## LOS 11.b: 빅데이터, 인공지능, 머신러닝

### 빅데이터의 3V — 숫자로 느끼기

상상해보세요. 당신이 애널리스트인데 상사가 "내일까지 글로벌 소비자 심리를 분석해"라고 한다.

**Volume (규모):** 트위터에서 하루에 생성되는 트윗이 약 5억 건이다. 1건당 30자만 잡아도 신문 수십억 장 분량이다. 사람이 읽을 수 있는가? 불가능하다.

데이터 단위의 진화:
- **메가바이트(MB)**: 책 1권 ≈ 1MB
- **기가바이트(GB)**: 영화 1편 ≈ 4GB
- **테라바이트(TB)**: 1,000GB. 미국 의회 도서관 ≈ 15TB
- **페타바이트(PB)**: 1,000TB. 구글이 하루 처리하는 데이터 ≈ 20PB

**왜 이것이 투자에 중요한가?** 2010년에는 테라바이트 단위의 데이터를 처리할 수 있는 펀드가 거의 없었습니다. 지금은 클라우드 컴퓨팅 덕분에 소규모 펀드도 페타바이트 급 데이터를 분석할 수 있습니다. 정보 우위의 장벽이 낮아지고 있습니다.

**Velocity (속도):** 같은 정보라도 1초 먼저 아는 사람이 돈을 번다. HFT 회사가 서버를 거래소 바로 옆에 두는 이유다.

숫자를 비교하자:
- 인간 애널리스트: ECB 총재 발언을 읽고 30분 후 분석 보고서 작성
- AI 시스템: 발언 시작 0.3초 만에 톤(긍정/부정) 분류 → 자동 매매

**왜 이런 결과가 나올까?** 30분 후에는 시장이 이미 반응했다. 0.3초에 반응해야 수익이 난다.

**지연시간(Latency) 개념:**
- **Low latency**: 실시간에 가까운 데이터. 주가 피드, 뉴스 와이어
- **High latency**: 시차가 있는 데이터. 분기 재무제표(3개월 지연), 센서스 데이터(연간)

| 데이터 소스 | 지연시간 | 투자 활용 |
|------------|---------|----------|
| 주가 실시간 피드 | 밀리초 | HFT, 마켓 메이킹 |
| 트위터 감성 | 초~분 | 이벤트 드리븐 전략 |
| 신용카드 매출 | 일~주 | 실적 예측 |
| 분기 재무제표 | 월 | 기본적 분석 |
| 인구 센서스 | 연 | 장기 거시 전략 |

**변수를 바꾸면?** 같은 "ECB 발언" 정보라도:
- 0.3초 안에 반응 → +0.5% 수익
- 30분 후 반응 → 0% (이미 가격에 반영)
- 다음 날 반응 → -0.2% (과잉 반응 후 되돌림에 매수)

속도가 아닌 **최적 타이밍**이 진짜 경쟁 우위인 경우도 있습니다.

**Variety (다양성):** 정형(스프레드시트), 반정형(HTML), 비정형(동영상, 소셜 미디어). 전체 데이터의 약 80%가 비정형이다.

| 데이터 유형 | 구조 수준 | 예시 | 분석 난이도 |
|-----------|---------|------|-----------|
| 정형(Structured) | 높음 | 스프레드시트, 데이터베이스 | 낮음 |
| 반정형(Semi-structured) | 중간 | HTML, JSON, 이메일 | 중간 |
| 비정형(Unstructured) | 낮음 | 동영상, 위성 사진, SNS | 높음 |

| 데이터 소스 | 예시 | 왜 유용한가? |
|------------|------|------------|
| 개인 생성 | 트위터, 제품 리뷰, 검색 기록 | 소비자 심리 실시간 파악 |
| 기업 배기가스(Corporate Exhaust) | 은행 거래, POS 데이터 | 공식 발표 전 실적 추정 |
| IoT | GPS, 공장 센서, RFID | 실물 경제 실시간 추적 |

**시나리오 — Corporate Exhaust로 실적 예측:**
대형마트의 POS(판매시점) 데이터를 분석합니다.
- 10월 첫 2주 매출: 전년 동기 대비 +12%
- 3분기 실적 발표: 11월 초
- 시장 컨센서스: +5% 성장 예상

POS 데이터를 가진 펀드는 실적 발표 3주 전에 "시장이 과소평가하고 있다"는 것을 알 수 있습니다. 이것이 **대안 데이터(alternative data)**의 가치입니다.

### 빅데이터의 도전 과제

빅데이터를 쓸 수 있다고 해서 무조건 좋은 것은 아닙니다:

1. **데이터 품질**: 이상치, 결측값, 표본 편향이 있으면 쓰레기가 들어가서 쓰레기가 나옴(GIGO)
2. **충분성과 적합성**: 데이터 양이 충분한지, 목적에 적합한지 확인 필요
3. **비정형 데이터 처리**: 텍스트, 이미지를 수치로 변환하는 과정 자체가 어려움
4. **비용**: 위성 사진 데이터 구독료 연 수백만 달러. ROI를 정당화할 수 있는가?

### 데이터 사이언스 5단계

당신이 원유를 발견했다고 합시다. 원유 자체로는 쓸모없다. 정유 공장을 거쳐야 휘발유가 된다. 빅데이터도 마찬가지다:

1. **Capture** (수집) — 데이터를 수집하고 사용 가능한 형태로 변환
2. **Curation** (큐레이션) — 데이터 품질 보증. 불량·결측 데이터 처리
3. **Storage** (저장) — 데이터 아카이빙 및 접근
4. **Search** (검색) — 저장된 데이터에서 필요한 정보 탐색
5. **Transfer** (전송) — 소스나 저장소에서 필요한 곳으로 이동

**시나리오 — 위성 사진으로 유조선 추적:**
1. **Capture**: 매일 전 세계 항만의 위성 사진 수집
2. **Curation**: 구름에 가려진 사진 필터링, 해상도 불량 이미지 제거
3. **Storage**: 수 페타바이트의 이미지를 클라우드에 저장
4. **Search**: "중동 항만에서 출발한 유조선 수" 쿼리
5. **Transfer**: 분석 결과를 트레이딩 시스템으로 실시간 전송

이 데이터로 "원유 공급량이 예상보다 많다 → 유가 하락 예상 → 원유 선물 매도" 전략을 구사합니다.

### 시각화 기법

정형 데이터는 익숙한 차트(막대, 선, 산점도)로 표현합니다. 비정형 데이터에는:
- **워드 클라우드(Word Cloud)**: 텍스트에서 단어 빈도를 시각화. 자주 나오는 단어가 크게 표시
- **마인드 맵(Mind Map)**: 개념 간 논리적 관계를 시각화

### 인공지능과 신경망

**인공지능(AI):** 인간의 인지 과정을 시뮬레이션하도록 프로그래밍된 컴퓨터 시스템.

**신경망(Neural Networks):** 인간 뇌의 정보 처리 방식을 모방한 AI의 한 형태. 입력층 → 은닉층 → 출력층으로 구성되며, 각 연결에 가중치를 부여하여 학습합니다.

**왜 신경망이 금융에서 중요한가?** 전통 통계 모델은 "변수 간 관계를 사전에 정의"합니다. 신경망은 "데이터가 알려줄 테니 관계를 알아서 찾아라"고 합니다. 인간이 예상하지 못한 비선형 패턴을 발견할 수 있습니다.

### 머신러닝 — 숫자로 비교

전통 통계와 머신러닝의 차이를 느껴보자.

당신이 "GDP 성장률로 주가 수익률을 예측하라"는 과제를 받았다.

**전통 통계 접근:** "둘 사이에 직선 관계일 것이다"라고 가정하고 회귀분석을 돌린다.
- GDP 3% → 주가 +8%. GDP 5% → 주가 +12%.

**머신러닝 접근:** "어떤 관계가 있는지 컴퓨터가 알아서 찾아라. 직선이든 곡선이든 상관없다."
- GDP 3% → 주가 +8%. GDP 5% → 주가 +15%. GDP 7% → 주가 +10% (비선형!)

**왜 이런 결과가 나올까?** 현실에서는 GDP가 너무 높으면 인플레이션과 금리 인상으로 주가가 오히려 떨어질 수 있다. 직선 모형은 이걸 잡지 못하지만 머신러닝은 잡을 수 있다.

**시나리오 — 채무 불이행 예측:**

전통 모델(로지스틱 회귀):
- 입력: 신용 점수, 소득, 부채 비율 (3개 변수)
- 정확도: 78%

머신러닝(랜덤 포레스트):
- 입력: 신용 점수, 소득, 부채 비율 + 거래 패턴, 소셜 미디어, 지역 실업률 (50개 변수)
- 정확도: 89%

**왜 이런 결과가 나올까?** 머신러닝은 변수 간 복잡한 상호작용을 포착합니다. "부채 비율이 높지만 + 소득이 꾸준히 증가 + 지역 실업률이 낮으면" 불이행 위험이 낮다는 복합 패턴을 학습합니다. 전통 모델은 이런 3중 상호작용을 잡기 어렵습니다.

**3단계 데이터 분할:** 시험 공부로 비유하면:
- **훈련 데이터(Training)** = 교과서 (패턴 학습)
- **검증 데이터(Validation)** = 모의시험 (하이퍼파라미터 조정)
- **테스트 데이터(Test)** = 실제 시험 (최종 성능 평가)

**왜 3단계인가?** 교과서로 공부하고 바로 실전 시험을 보면, "얼마나 잘 공부했는지" 확인할 기회가 없습니다. 모의시험이 그 중간 점검 역할을 합니다. 검증 데이터는 모델의 하이퍼파라미터(학습률, 층 수 등)를 조정하는 데 사용됩니다.

| 유형 | 특징 | 금융 예시 |
|------|------|---------|
| **지도학습(Supervised)** | 정답 레이블 있음 | 채무 불이행 예측 (불이행 = 1, 정상 = 0) |
| **비지도학습(Unsupervised)** | 정답 없이 패턴 발견 | 고객 군집화 (비슷한 투자 성향끼리 그룹) |
| **딥러닝(Deep Learning)** | 다층 신경망 | 10-K 보고서 감성 분석, 차트 패턴 인식 |

**지도학습 vs 비지도학습 — 비유:**
- 지도학습: 선생님이 "이 사과는 빨강, 이 바나나는 노랑"이라고 알려준 후, 새 과일의 색깔을 맞추게 함
- 비지도학습: 과일을 주고 "너 스스로 비슷한 것끼리 분류해봐"라고 함. 색깔, 크기, 모양 등 다양한 기준으로 그룹을 만들 수 있음

### 과적합과 과소적합 — 두 학생의 이야기

**학생 A (과적합):** 기출 문제 500개를 통째로 외웠다. 기출 시험이면 100점이다. 하지만 새로운 문제가 나오면 0점.

**학생 B (과소적합):** 교과서 목차만 읽었다. 기출이든 새 문제든 50점.

| 문제 | 훈련 성적 | 실전 성적 |
|------|---------|---------|
| 과적합 (학생 A) | 99% | 50% |
| 과소적합 (학생 B) | 60% | 55% |
| **적정 학습** | 85% | 80% |

**왜 이런 결과가 나올까?** 학생 A는 "월요일에 비가 오면 주가가 올랐다"는 우연의 패턴까지 법칙으로 학습한 것이다. 학생 B는 진짜 패턴조차 학습하지 않은 것이다. 적정 학습은 "진짜 패턴만 학습하고 노이즈는 무시"하는 것입니다.

**숫자 시나리오 — 과적합의 실제 위험:**
헤지펀드가 15년간 주가 데이터로 모델을 만들었습니다.
- 훈련 기간(2005-2019) 수익률: 연 +25%
- 테스트 기간(2020-2022) 수익률: 연 -8%
- 원인: 모델이 2008년 금융위기와 2020년 코로나 같은 극단적 이벤트의 "사후 패턴"을 학습. 하지만 다음 위기는 다른 형태로 올 것.

**만약 훈련 데이터를 10배로 늘리면?** 과적합이 줄어든다. 우연의 패턴이 많은 데이터에서는 반복되지 않기 때문이다.

**과적합을 줄이는 방법들:**
1. **더 많은 데이터** — 우연의 패턴이 희석됨
2. **교차 검증(Cross-validation)** — 데이터를 여러 번 나눠 반복 테스트
3. **정규화(Regularization)** — 모델이 너무 복잡해지지 않도록 패널티 부여
4. **변수 축소** — 불필요한 변수를 제거하여 노이즈 감소

**블랙박스 문제:** 딥러닝이 "이 주식을 사라"고 했는데 **왜** 그런지 사람이 이해할 수 없다. 규제 환경에서 "AI가 그렇게 판단했습니다"라고 답할 수 없다.

**시나리오 — 규제 충돌:**
은행의 AI 모델이 대출 신청을 거부했습니다. 신청자가 "왜 거부했나요?"라고 물었을 때:
- 전통 모델: "소득 대비 부채 비율이 45%로 기준(40%)을 초과" → 설명 가능
- 딥러닝 모델: "50개 변수의 복합적 상호작용으로 판단" → 설명 불가

유럽의 GDPR은 "설명 가능한 AI(Explainable AI)"를 요구합니다. 이것이 금융에서 딥러닝 적용의 현실적 제약입니다.

## LOS 11.c: 투자 관리에서의 활용

### 텍스트 분석 — 숫자로 비교

당신이 ECB 총재 기자회견을 모니터링한다고 합시다.

- **3월 발언:** "경제 성장이 견조하다" → 긍정 단어 15개, 부정 단어 3개. 감성 점수 +0.67
- **6월 발언:** "하방 위험이 상당하다" → 긍정 단어 5개, 부정 단어 12개. 감성 점수 -0.41

**왜 이런 결과가 나올까?** 감성 점수가 +0.67에서 -0.41로 급락했다. 중앙은행이 비둘기파로 돌아선 것이다 → 금리 인하 가능성 → 채권 매수 신호.

**시나리오 — 10-K 보고서 분석:**
기업의 연차보고서(10-K)에서 특정 단어의 빈도 변화를 추적합니다.

| 단어/구절 | 2022년 빈도 | 2023년 빈도 | 변화 | 해석 |
|---------|-----------|-----------|------|------|
| "불확실성" | 12 | 28 | +133% | 경영진 우려 증가 |
| "성장 기회" | 15 | 8 | -47% | 확장 의지 감소 |
| "소송" | 3 | 11 | +267% | 법적 리스크 급증 |
| "강건한" | 20 | 22 | +10% | 큰 변화 없음 |

이 분석만으로도 "이 기업이 어려움에 직면하고 있다"는 조기 경보를 얻을 수 있습니다. 실적 발표 전에 말이죠.

**변수를 바꾸면?** 같은 분석을 산업 전체(예: S&P 500 기업 전부)에 적용하면, 산업별 심리 지수를 구성할 수 있습니다. "기술 섹터의 불확실성 언급이 금융 섹터보다 3배 빠르게 증가 중" 같은 인사이트를 얻습니다.

### 자연어 처리 (NLP)

텍스트 분석이 "단어를 세는 것"이라면, NLP는 "문맥을 이해하는 것"이다.

두 문장을 비교하자:
- "이 회사 실적이 죽인다" → NLP는 이것이 **긍정**임을 안다
- "이 회사 실적이 죽었다" → NLP는 이것이 **부정**임을 안다

단순 단어 빈도 분석은 둘 다 "죽"이라는 부정 단어로 처리할 수 있다. NLP는 문맥을 파악한다.

**추가 예시:**
- "이 펀드의 수익률은 나쁘지 않다" → 단어 빈도: "나쁘다" = 부정. NLP: 부정의 부정 = **긍정**
- "이전만큼 좋지는 않다" → NLP: **약한 부정** (이전보다 나빠졌지만 완전히 나쁜 건 아님)

**금융에서의 NLP 활용:**
1. **규제 준수**: 직원 커뮤니케이션을 모니터링하여 내부자 거래 징후 탐지
2. **리서치 보고서 분석**: 수천 개의 애널리스트 보고서에서 미묘한 감성 변화 감지
3. **실적 발표 콜 분석**: CEO의 어조, 망설임, 답변 회피 패턴을 감지하여 실적 신뢰도 평가

**시나리오 — 실적 발표 콜 분석:**
CEO가 분석가 질문에 답할 때:
- 평소: 직접적, 구체적 수치 언급, 자신감 있는 어조
- 이번 분기: 추상적 표현 증가, "불확실성" 반복, 긴 침묵

NLP 모델이 이 변화를 감지하고 "경영진 신뢰도 하락" 신호를 발생시킵니다. 실제 실적이 나쁘지 않았어도, 향후 실적 하향의 조기 경보가 될 수 있습니다.

### 리스크 거버넌스

기업은 다양한 리스크에 노출됩니다. 금융 규제당국은 리스크 평가와 스트레스 테스트를 요구합니다.

**왜 빅데이터가 필요한가?** 시뮬레이션, 시나리오 분석, 기타 리스크 분석 기법은 대량의 정량적 데이터와 정성적 정보를 필요로 합니다. 머신러닝과 빅데이터 기술은 특히 실시간 데이터를 사용하여 리스크 노출을 모니터링할 때 유용합니다.

**시나리오 — 실시간 리스크 모니터링:**
전통적 리스크 관리: 월말에 포트폴리오 VaR을 계산
빅데이터 리스크 관리: 실시간으로 시장 데이터 + 뉴스 + SNS를 모니터링하여 급격한 변동 징후를 조기 감지

2020년 3월 코로나 폭락:
- 전통 모델: 2월 말 VaR 보고서는 "정상 범위" → 3월 폭락에 무방비
- 실시간 AI 모델: 1월부터 중국 SNS의 "바이러스" 언급 급증 → 2월 초 경보 발생 → 헤지 포지션 구축 시간 확보

### 알고리즘 트레이딩

당신이 1,000만 주를 사야 한다고 합시다. 한 번에 다 사면? 가격이 폭등한다. 알고리즘이 하루 동안 100번에 나눠서, 거래량이 많은 시간에 더 많이 산다.

**왜 알고리즘이 인간보다 나은가?**

| 기준 | 인간 트레이더 | 알고리즘 |
|-----|-----------|---------|
| 실행 속도 | 초~분 | 밀리초 |
| 감정 | 공포, 탐욕 영향 | 없음 |
| 일관성 | 피로, 실수 가능 | 24/7 동일 성능 |
| 시장 충격 | 대형 주문 시 가격 이동 | 최적 분할로 충격 최소화 |

**시나리오 — 대형 주문 실행:**
1,000만 주를 사야 합니다. 일평균 거래량이 500만 주인 종목입니다.

- **한 번에 매수**: 수요 폭증 → 가격 5% 상승 → 평균 매수가 높아짐. 추가 비용 ≈ $5M
- **TWAP(시간 가중) 알고리즘**: 하루 동안 균등 분배 → 가격 영향 1% → 추가 비용 ≈ $1M
- **VWAP(거래량 가중) 알고리즘**: 거래량 많은 시간에 집중 → 가격 영향 0.5% → 추가 비용 ≈ $0.5M

알고리즘이 $4.5M을 절약했습니다.

**HFT (High-Frequency Trading):**
- 밀리초 단위 거래
- 미국 주식 거래량의 약 50%
- 시장 간 미세한 가격 차이를 이용(차익거래)
- 건당 이익은 매우 작지만(0.001% 수준), 하루 수백만 건 거래로 수익 축적

**변수를 바꾸면?** HFT가 없어지면:
- 스프레드(매수-매도 호가 차이)가 넓어짐 → 일반 투자자의 거래 비용 증가
- 시장 간 가격 불일치가 오래 지속 → 시장 효율성 감소
- 반면: 플래시 크래시(순간 폭락) 위험 감소

## 핵심 개념 정리 (Key Concepts)

### LOS 11.a
- 핀테크 = 금융에 적용되는 기술. 핀테크 기업 = 금융 기술을 개발하는 회사
- 핵심 발전: (1) 대규모 다양한 데이터 처리 능력 (2) AI/ML 분석 도구
- ATM → 온라인 뱅킹 → 알고 트레이딩 → 로보어드바이저 → AI 신용평가의 진화

### LOS 11.b
- 빅데이터 3V: Volume(규모), Velocity(속도), Variety(다양성)
- 데이터 소스: 개인 생성, 기업 배기가스(corporate exhaust), IoT
- 데이터 사이언스 5단계: Capture → Curation → Storage → Search → Transfer
- 시각화: 정형 = 차트/그래프, 비정형 = 워드 클라우드/마인드 맵
- AI = 인간 인지 시뮬레이션. 신경망 = AI의 한 형태
- 머신러닝: 지도학습(레이블 있음)/비지도학습(레이블 없음)/딥러닝(다층 신경망)
- 데이터 분할: 훈련(학습) → 검증(튜닝) → 테스트(최종 평가)
- 과적합: 노이즈까지 학습 → 새 데이터 실패. 과소적합: 너무 단순하여 패턴 미학습
- 블랙박스 문제: 결과는 나오지만 이유를 설명할 수 없음

### LOS 11.c
- 텍스트 분석: 단어 빈도, 감성 분석. 비정형 텍스트/음성을 정량화
- NLP: 문맥 이해. 음성 인식, 언어 번역, 미묘한 감성 변화 감지
- 리스크 거버넌스: 실시간 데이터로 리스크 노출 모니터링
- 알고리즘 트레이딩: 사전 설정 규칙 기반 자동 매매. 최적 주문 실행, HFT
