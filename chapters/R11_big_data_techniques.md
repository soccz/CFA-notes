## 왜 핀테크와 빅데이터가 중요한가?

> **[Level 1 — 비유]** 과거의 투자 분석가는 도서관에서 책 몇 권만 읽고 판단했습니다. 오늘날의 분석가는 전 세계 모든 도서관, 소셜 미디어, 위성 사진, 신용카드 거래 내역까지 한꺼번에 볼 수 있습니다. 문제는 "이 엄청난 양의 정보에서 어떻게 의미 있는 신호를 추출하느냐"입니다. 이것이 빅데이터와 머신러닝의 역할입니다.

> **[Level 2 — 개념]** 과거에는 금융 분석가들이 재무제표와 가격 데이터만 다뤘습니다. 오늘날은 소셜 미디어, 위성 이미지, 신용카드 거래 데이터 등 방대한 양의 새로운 정보가 쏟아집니다. 이것들을 처리하고 분석하는 도구와 기술의 발전이 **핀테크(FinTech)** 입니다.

> **[개념 연결 ← R06, R10]** R06에서 배운 시뮬레이션과 R10에서 배운 회귀분석은 전통적인 분석 도구입니다. 이 챕터에서 소개하는 머신러닝은 전통적 회귀분석의 한계(비선형 관계, 고차원 데이터)를 극복하는 차세대 도구입니다. 하지만 기본 원리(과적합 vs 과소적합, 데이터 품질의 중요성)는 동일합니다.

## LOS 11.a: 핀테크 (FinTech)

**핀테크(FinTech)**란 금융 서비스 산업에 적용될 수 있는 기술 발전을 의미합니다. 핀테크 기업은 금융업을 위한 기술을 개발하는 회사입니다.

**핀테크의 주요 발전 영역:**
1. 다양한 소스와 형태의 대규모 데이터 처리 기능 향상
2. 방대한 데이터셋 분석을 위한 인공지능 등의 도구와 기법

---

## LOS 11.b: 빅데이터, 인공지능, 머신러닝

### 빅데이터 (Big Data)

**정의:** 경제 내에서 생성되는 잠재적으로 유용한 모든 정보.

**전통적 데이터 소스:** 금융시장, 기업 재무보고서, 정부 경제 통계

**비전통적(대안) 데이터 소스:**
- **개인 생성 데이터**: 소셜 미디어 게시물, 온라인 리뷰, 이메일, 웹사이트 방문 기록
- **기업 생성 데이터 (Corporate Exhaust)**: 은행 기록, 소매점 스캐너 데이터
- **사물인터넷 (IoT, Internet of Things)**: 스마트폰, 스마트 빌딩 등에 내장된 센서(RFID 칩 등)들의 네트워크

---

### 빅데이터의 3V 특성

| 특성 | 설명 |
|------|------|
| **Volume (규모)** | 데이터 양이 기하급수적으로 증가: 메가바이트 → 기가바이트 → 테라바이트(1,000GB) → 페타바이트(1,000TB) |
| **Velocity (속도)** | 데이터 전달 속도. 실시간 주가 피드 = **낮은 지연(low latency)**, 주기적/시차 있는 데이터 = **높은 지연(high latency)** |
| **Variety (다양성)** | 구조화 정도의 범위: 정형(스프레드시트/DB) → 반정형(사진/웹코드) → 비정형(동영상) |

---

### 데이터 사이언스 (Data Science)

빅데이터에서 정보를 추출하는 방법론.

**데이터 처리 단계:**
1. **Capture (수집)**: 데이터 수집 및 사용 가능한 형태로 변환
2. **Curation (큐레이션)**: 불량/결측 데이터 조정으로 품질 보증
3. **Storage (저장)**: 데이터 보관 및 접근
4. **Search (검색)**: 필요한 정보 탐색
5. **Transfer (전송)**: 소스에서 필요한 곳으로 이동

**시각화 기법:**
- 정형 데이터: 차트, 그래프
- 비정형 데이터: 워드 클라우드(단어 빈도 시각화), 마인드맵(개념 간 논리 관계)

**빅데이터 활용의 도전 과제:**
- 데이터 품질 보증 (이상치, 결측값, 표본 편향)
- 충분한 데이터 양 확보
- 정성적/비정형 데이터의 전처리 (인공지능 적용 가능)

---

### 인공지능 (Artificial Intelligence, AI)

**정의:** 인간의 인지를 모방하도록 프로그래밍된 컴퓨터 시스템.

**신경망 (Neural Networks):**
- 인간 뇌처럼 정보를 처리하도록 프로그래밍된 AI의 대표적 예
- 특히 정성적·비정형 데이터를 전처리하고 구조화하는 데 유용
- 딥러닝의 기반 구조

---

### 머신러닝 (Machine Learning)

**정의:** 인간의 도움 없이 스스로 학습하는 컴퓨터 알고리즘. 입력 데이터(확률분포 가정 없음)를 받아 출력 데이터를 모델링하거나 패턴을 탐지.

**머신러닝 프로세스:**
```
훈련 데이터셋     →  검증 데이터셋     →  테스트 데이터셋
(패턴 학습)           (모델 정교화)         (예측 능력 평가)
```

**머신러닝 유형:**

| 유형 | 특징 |
|------|------|
| **지도학습 (Supervised Learning)** | 입력/출력 데이터가 레이블됨. 기계가 입력→출력 모델링 후 새 데이터에 적용 |
| **비지도학습 (Unsupervised Learning)** | 입력 데이터 레이블 없음. 기계가 데이터 구조 스스로 파악 |
| **딥러닝 (Deep Learning)** | 여러 층(layer)의 신경망을 사용하여 패턴 식별. 단순 패턴 → 복잡 패턴 순서로 계층적 학습. 지도/비지도 학습 모두 활용 가능. 이미지 인식, 음성 인식이 대표적 응용 사례 |

---

### 과적합과 과소적합

| 문제 | 설명 | 결과 |
|------|------|------|
| **과적합 (Overfitting)** | 노이즈까지 학습, 너무 정확하게 훈련 데이터에 맞춤 | 모델이 너무 복잡, 새 데이터에서 성능 저하 |
| **과소적합 (Underfitting)** | 실제 패턴을 노이즈로 처리, 충분히 학습 못함 | 모델이 너무 단순, 데이터 설명 불충분 |

**블랙박스 문제:** 머신러닝 결과가 설명하기 어려운 관계에 기반하는 경우 — 어떻게 그 결론에 도달했는지 알기 어려움.

## LOS 11.c: 투자 관리에서의 빅데이터/데이터 사이언스 활용

### 1. 텍스트 분석 (Text Analytics)

**정의:** 텍스트 또는 음성 형태의 비정형 데이터 분석.

**활용 예시:**
- 단어/구문 빈도 분석
- 기업 규제 보고서(SEC 등) 자동 평가
- 실적 발표 문서의 감성 분석

---

### 2. 자연어 처리 (Natural Language Processing, NLP)

**정의:** 인간 언어를 해석하기 위한 컴퓨터와 AI 활용.

**활용 예시:**
- 음성 인식, 언어 번역
- 직원 통신에서 규제 준수 여부 확인
- 대량의 리서치 보고서 분석으로 애널리스트 추천 이면의 감성 변화 탐지

---

### 3. 위험 거버넌스 (Risk Governance)

**정의:** 기업의 다양한 위험 노출을 이해하고 관리.

**활용 예시:**
- 금융 규제당국이 요구하는 리스크 평가 및 스트레스 테스트
- 머신러닝으로 실시간 데이터 기반 위험 노출 모니터링
- 시뮬레이션, 시나리오 분석

---

### 4. 알고리즘 트레이딩 (Algorithmic Trading)

**정의:** 사전 정의된 규칙에 기반한 컴퓨터화된 증권 거래.

**활용 예시:**
- 실시간 가격/거래량 데이터로 최적 주문 실행
- 대규모 주문을 여러 거래소에 최적으로 분산
- **초고빈도 거래(HFT, High-Frequency Trading)**: 장 중 증권 가격 오류를 찾아 활용

## Module Quiz 11.1

**Q1.** 핀테크를 가장 정확하게 설명한 것은?

A. 기술을 금융 서비스 산업에 적용하는 것
B. 정부 발행 화폐를 전자 화폐로 대체하는 것
C. 분산 원장 기술로 증권 거래를 청산/결제하는 것

**정답: A** — 핀테크 = 금융 서비스에 적용되는 기술 발전. B(암호화폐)와 C(블록체인)는 핀테크의 일부 예시일 뿐

---

**Q2.** 빅데이터 분석에 가장 유용한 기술 발전은?

A. 머신러닝
B. 높은 지연(high-latency) 수집
C. 사물인터넷 (IoT)

**풀이:**
- A. **정답** — 머신러닝은 대용량 데이터에서 패턴을 식별하고 모델링하는 데 유용
- B. 높은 지연은 데이터 분석에 불리 (실시간 처리 불가)
- C. IoT는 빅데이터의 소스(원천)이지, 분석 도구가 아님

**정답: A**
