# Reading 11: Introduction to Big Data Techniques

> **핵심 질문:** 디지털 시대에 금융 데이터는 어떻게 변하고 있으며, 인공지능과 머신러닝이 투자에 어떻게 활용되는가?

---

## 왜 핀테크와 빅데이터가 중요한가?

당신이 주식을 사려고 한다고 하자. 20년 전이라면 기업의 재무제표를 읽고, 뉴스를 찾아보는 정도가 전부였다. 하지만 지금은 어떤가?

어떤 헤지펀드는 **위성 사진**으로 월마트 주차장에 차가 몇 대 있는지 세서 매출을 예측한다. 또 어떤 펀드는 **소셜 미디어**에서 특정 브랜드에 대한 감성(긍정/부정)을 실시간으로 분석한다. 심지어 **신용카드 거래 데이터**를 사서 소비자 지출 패턴을 추적하기도 한다.

문제는 이 데이터의 양이 인간이 처리할 수 있는 범위를 한참 넘어선다는 것이다. 하루에 생성되는 데이터가 수십 페타바이트에 달한다. 사람이 일일이 읽을 수 없으니, **컴퓨터가 대신 패턴을 찾아주는 기술** — 즉 머신러닝과 인공지능 — 이 필요해진 것이다.

이것이 **핀테크(FinTech)** 의 핵심이다: "금융에서 기술이 어떻게 쓰이는가?"

> **개념 연결 ← R06, R10:** R06에서 배운 시뮬레이션과 R10에서 배운 회귀분석은 전통적인 분석 도구다. 이 챕터에서 소개하는 머신러닝은 전통적 회귀분석의 한계(비선형 관계, 고차원 데이터)를 극복하는 차세대 도구다. 하지만 기본 원리(과적합 vs 과소적합, 데이터 품질의 중요성)는 동일하다.

## LOS 11.a: 핀테크 (FinTech)

**핀테크(FinTech)**란 금융 서비스 산업에 적용될 수 있는 기술 발전을 의미한다.

이것을 왜 별도로 정의하느냐? 금융은 원래부터 기술 의존도가 높은 산업이기 때문이다. ATM, 온라인 뱅킹, 전자 주식 거래 — 이것들이 모두 핀테크다. 최근의 핀테크는 두 가지 방향에서 진화하고 있다:

1. **데이터 처리 능력의 폭발적 향상:** 과거에는 스프레드시트 하나로 충분했던 분석이, 이제는 수십억 건의 거래 데이터를 실시간으로 처리해야 한다.
2. **분석 도구의 진화:** 단순 통계에서 인공지능·머신러닝으로. 사람이 못 보는 패턴을 기계가 찾아낸다.

## LOS 11.b: 빅데이터, 인공지능, 머신러닝

### 빅데이터 (Big Data)

**정의:** 경제 내에서 생성되는 잠재적으로 유용한 모든 정보.

전통적으로 금융 분석가가 쓰던 데이터는 **주가, 재무제표, 경제 통계** 정도였다. 이것들은 숫자가 깔끔하게 정리된 정형 데이터(structured data)다.

그런데 요즘은 전혀 다른 종류의 데이터가 투자에 쓰인다:

| 데이터 소스 | 예시 | 왜 유용한가? |
|------------|------|------------|
| **개인 생성 데이터** | 트위터 게시물, 제품 리뷰, 이메일, 웹 검색 기록 | 소비자 심리를 실시간으로 파악 가능. "아이폰 불만 트윗이 급증하면 애플 실적이 나빠질 수 있다" |
| **기업 배기가스(Corporate Exhaust)** | 은행 거래 기록, 편의점 POS 스캐너 데이터 | 기업의 공식 발표 전에 실적을 추정 가능. "스타벅스 카드 결제 데이터로 이번 분기 매출 예측" |
| **사물인터넷(IoT)** | 스마트폰 GPS, 공장 센서, RFID 칩 | 실물 경제 활동을 실시간으로 추적. "중국 공장 가동률을 센서 데이터로 추정" |

---

### 빅데이터의 3V 특성

빅데이터를 "빅"하게 만드는 것은 단순히 양이 많다는 것만이 아니다. 세 가지 차원에서 "크다":

**Volume (규모):** 데이터의 절대적 양. 1998년에 구글 전체 인덱스가 26MB였다. 지금은 하루에 그 수십만 배의 데이터가 생성된다. 메가바이트 → 기가바이트 → 테라바이트(1,000GB) → 페타바이트(1,000TB) 순서로 단위가 커진다.

**Velocity (속도):** 데이터가 생성되고 전달되는 속도.
- **낮은 지연(low latency):** 실시간 주가 피드처럼 즉시 도착하는 데이터. 초고빈도 거래(HFT)에 필수.
- **높은 지연(high latency):** 분기 재무제표처럼 시차를 두고 공개되는 데이터.

왜 속도가 중요한가? 같은 정보라도 **1초 먼저 아는 사람**이 돈을 번다. 뉴스가 나온 후 0.001초 만에 거래하는 HFT 회사가 존재하는 이유다.

**Variety (다양성):** 데이터의 구조화 정도.
- **정형(Structured):** 스프레드시트처럼 행과 열로 정리된 데이터. 주가, 재무비율.
- **반정형(Semi-structured):** 어느 정도 구조는 있지만 완전하지 않은 데이터. 웹 HTML 코드, 사진 메타데이터.
- **비정형(Unstructured):** 구조가 전혀 없는 데이터. 동영상, 음성, 소셜 미디어 글. 전체 데이터의 약 80%가 비정형이다.

### 데이터 사이언스 (Data Science)

빅데이터가 원유라면, 데이터 사이언스는 정유 공장이다. 원유 자체로는 쓸모없지만, 정제하면 가솔린이 된다.

**데이터 처리 5단계:**

| 단계 | 비유 | 실제 작업 |
|------|------|----------|
| 1. **Capture (수집)** | 원유 채굴 | 다양한 소스에서 데이터 수집, 사용 가능한 형태로 변환 |
| 2. **Curation (큐레이션)** | 불순물 제거 | 불량·결측 데이터 처리. "이 셀이 빈칸인데 0인가, 아니면 데이터가 없는 건가?" |
| 3. **Storage (저장)** | 저장 탱크 | 데이터 보관 및 접근 체계 구축 |
| 4. **Search (검색)** | 필요한 성분 추출 | "2023년 한국 소비자의 스마트폰 구매 패턴 데이터만 찾아줘" |
| 5. **Transfer (전송)** | 파이프라인 운송 | 데이터를 필요한 시스템으로 전달 |

**시각화 기법:**
- 정형 데이터 → 차트, 그래프 (우리가 익숙한 방식)
- 비정형 데이터 → **워드 클라우드**(단어 빈도 시각화: 자주 나오는 단어일수록 크게 표시), **마인드맵**(개념 간 관계를 시각화)

**빅데이터 활용의 현실적 어려움:**
빅데이터가 만능은 아니다. 쓰레기를 넣으면 쓰레기가 나온다(Garbage In, Garbage Out). 이상치, 결측값, 표본 편향이 있으면 아무리 좋은 알고리즘도 엉뚱한 결론을 낸다. 비정형 데이터(영상, 텍스트)를 분석 가능한 형태로 바꾸는 전처리 작업도 큰 과제다.

### 인공지능 (Artificial Intelligence, AI)

**정의:** 인간의 인지를 모방하도록 프로그래밍된 컴퓨터 시스템.

"인간의 인지를 모방"이란 무슨 뜻인가? 사람은 고양이 사진을 보면 "고양이"라고 안다. 숫자 패턴을 보면 "이건 사기 거래 같은데"라고 느낀다. AI는 이런 인식·판단·학습 능력을 컴퓨터로 구현하려는 시도다.

**신경망 (Neural Networks):**
인간의 뇌는 수십억 개의 뉴런이 서로 연결되어 정보를 처리한다. 신경망은 이 구조를 모방한 것이다. 입력 데이터 → 여러 층(layer)의 처리 → 출력 결과. 특히 비정형 데이터(이미지, 텍스트)를 분석 가능한 형태로 바꾸는 데 탁월하다. "이 뉴스 기사가 긍정적인가 부정적인가?"를 판단하는 것이 대표적 사례다.

### 머신러닝 (Machine Learning)

**정의:** 인간의 도움 없이 스스로 학습하는 컴퓨터 알고리즘.

전통적 통계(예: 회귀분석)와의 핵심 차이: 전통 통계는 "데이터가 정규분포를 따른다"는 등의 **가정**을 세우고 시작한다. 머신러닝은 가정 없이 데이터 자체에서 패턴을 찾는다.

비유로 설명하면: 전통 통계는 "내가 생각하기에 X와 Y는 직선 관계일 것이다. 데이터로 확인해보자." 머신러닝은 "X와 Y 사이에 어떤 관계가 있는지 네(컴퓨터)가 알아서 찾아라."

**머신러닝 프로세스:**

```
훈련 데이터셋     →  검증 데이터셋     →  테스트 데이터셋
(패턴 학습)           (모델 정교화)         (예측 능력 평가)
```

왜 3단계로 나누는가? 시험 공부로 비유하면:
- **훈련 데이터** = 교과서로 공부하는 것
- **검증 데이터** = 모의시험으로 자기 실력을 확인하고 부족한 부분을 보충하는 것
- **테스트 데이터** = 실제 시험. 처음 보는 문제를 얼마나 잘 푸는가?

교과서 문제만 100점 받고 실제 시험에서 0점이면 의미가 없다. 이것이 바로 **과적합(overfitting)** 문제다.

**머신러닝 유형:**

| 유형 | 비유 | 특징 |
|------|------|------|
| **지도학습 (Supervised)** | 선생님이 정답을 알려주면서 가르침 | 입력/출력 데이터가 모두 레이블됨. "이 이메일은 스팸이다/아니다"를 컴퓨터에게 수천 번 보여주면, 새 이메일이 스팸인지 스스로 판단 |
| **비지도학습 (Unsupervised)** | 정답 없이 학생이 스스로 패턴을 발견 | 입력 데이터만 있고 레이블 없음. 고객 데이터에서 "비슷한 고객 그룹"을 자동으로 분류(클러스터링) |
| **딥러닝 (Deep Learning)** | 초등학교 → 중학교 → 고등학교처럼 단계적으로 심화 학습 | 여러 층의 신경망으로 단순 패턴 → 복잡 패턴을 계층적으로 학습. 이미지 인식("이 사진은 고양이"), 음성 인식("시리야, 날씨 알려줘")이 대표 사례 |

---

### 과적합과 과소적합

이것은 머신러닝에서 가장 중요한 개념이다.

**과적합 (Overfitting):** 교과서 문제는 만점인데 실전에서 0점인 학생.

모델이 훈련 데이터의 노이즈(무의미한 변동)까지 외워버린 상태다. 예를 들어 과거 10년간 "월요일에 비가 오면 주가가 올랐다"는 패턴을 발견했다고 하자. 이것은 우연일 가능성이 높다. 하지만 과적합된 모델은 이것을 "법칙"으로 학습해버린다. 결과: 새로운 데이터에서는 성능이 급격히 떨어진다.

**과소적합 (Underfitting):** 교과서도 제대로 안 읽고 시험을 본 학생.

모델이 너무 단순해서 데이터에 존재하는 실제 패턴조차 잡아내지 못한다. "GDP가 올라가면 주가도 올라간다" — 이 정도만 학습하고 끝나는 것.

| 문제 | 증상 | 원인 |
|------|------|------|
| **과적합** | 훈련 데이터: 정확도 99%, 테스트 데이터: 정확도 50% | 모델이 너무 복잡. 노이즈까지 학습 |
| **과소적합** | 훈련 데이터: 정확도 60%, 테스트 데이터: 정확도 55% | 모델이 너무 단순. 실제 패턴도 못 잡음 |

**블랙박스 문제:** 머신러닝의 또 다른 한계. 딥러닝 모델이 "이 주식을 사라"고 결론을 내렸는데, **왜** 그 결론에 도달했는지 사람이 이해할 수 없는 경우가 많다. 이것은 규제 환경에서 큰 문제다. "왜 이 고객의 대출을 거절했는가?"에 "AI가 그렇게 판단했습니다"라고 답할 수 없기 때문이다.

## LOS 11.c: 투자 관리에서의 빅데이터/데이터 사이언스 활용

### 1. 텍스트 분석 (Text Analytics)

**정의:** 텍스트 또는 음성 형태의 비정형 데이터 분석.

실제 사례: 2018년 J.P. Morgan은 유럽중앙은행(ECB) 총재의 기자회견 발언을 실시간으로 텍스트 분석하여, 발언 톤이 긍정적이면 유로화 매수, 부정적이면 매도하는 전략을 사용했다. 사람이 기자회견을 듣고 판단하는 데 몇 분이 걸리지만, 텍스트 분석 알고리즘은 **밀리초** 만에 처리한다.

**활용 예시:**
- 단어/구문 빈도 분석: "기업 실적 발표에서 '불확실성'이란 단어가 전 분기보다 3배 많이 나왔다 → 부정적 신호"
- 기업 규제 보고서(SEC 등) 자동 평가: 수천 페이지 보고서를 사람이 다 읽을 수 없으니 AI가 핵심만 추출
- 실적 발표 문서의 감성 분석: "경영진의 톤이 지난 분기보다 방어적이다 → 실적 악화 가능성"

---

### 2. 자연어 처리 (Natural Language Processing, NLP)

**정의:** 인간 언어를 해석하기 위한 컴퓨터와 AI 활용.

텍스트 분석이 "단어를 세는 것"이라면, NLP는 "문맥을 이해하는 것"이다. "이 회사 실적이 죽인다"라는 문장에서 "죽인다"가 긍정인지 부정인지를 판단하는 것이 NLP의 영역이다.

**활용 예시:**
- 음성 인식, 언어 번역: 전 세계 시장의 뉴스를 실시간으로 자국어로 번역하여 분석
- 규제 준수 모니터링: 직원 이메일과 채팅에서 내부자 거래 의심 키워드를 자동 탐지
- 리서치 보고서 분석: 수천 명 애널리스트의 보고서를 NLP로 분석하여 전체적인 시장 심리 변화 추적

---

### 3. 위험 거버넌스 (Risk Governance)

**정의:** 기업의 다양한 위험 노출을 이해하고 관리.

2008년 금융위기 이후 규제당국은 금융기관에 훨씬 정교한 리스크 관리를 요구하게 되었다. 문제는 현대 금융기관의 위험이 너무 복잡하다는 것이다. 수만 개 포지션이 서로 얽혀 있고, 시장 상황이 실시간으로 바뀐다.

**활용 예시:**
- 스트레스 테스트: "만약 금리가 3% 급등하면 우리 포트폴리오 손실은?" — 과거에는 수일 걸렸던 시뮬레이션을 머신러닝으로 실시간 처리
- 실시간 위험 모니터링: 수만 개 포지션의 위험 노출을 매 순간 업데이트
- 시나리오 분석: "코로나 같은 팬데믹이 다시 오면?"에 대한 시뮬레이션

---

### 4. 알고리즘 트레이딩 (Algorithmic Trading)

**정의:** 사전 정의된 규칙에 기반한 컴퓨터화된 증권 거래.

당신이 1,000만 주를 사야 한다고 하자. 한 번에 사면 가격이 폭등한다(시장 충격). 그래서 알고리즘이 "하루 동안 100번에 나눠서, 거래량이 많은 시간에 더 많이, 적은 시간에 더 적게 산다"는 전략을 자동 실행한다.

**활용 예시:**
- 최적 주문 실행: 대규모 주문을 여러 거래소에 최적으로 분산하여 시장 충격 최소화
- **초고빈도 거래(HFT, High-Frequency Trading):** 밀리초(1/1000초) 단위로 거래. 뉴욕증권거래소와 시카고 옵션거래소 사이의 미세한 가격 차이를 0.001초 만에 포착하여 차익 실현. 현재 미국 주식 거래량의 약 50%가 HFT.

## 핵심 개념 정리 (Key Concepts)

### LOS 11.a 요약
핀테크 = 금융 서비스 산업에 적용되는 기술 발전. 데이터 처리 능력 향상 + AI/ML 분석 도구의 진화.

### LOS 11.b 요약
- **빅데이터:** 전통/비전통 소스의 방대한 정보. 3V 특성 — Volume(규모), Velocity(속도), Variety(다양성)
- **AI:** 인간 인지 모방 시스템. 신경망이 핵심 구현 방식
- **머신러닝:** 가정 없이 데이터에서 스스로 패턴을 찾는 알고리즘
  - 지도학습(정답 있음) / 비지도학습(정답 없음) / 딥러닝(다층 신경망)
- **과적합:** 모델이 너무 복잡 → 훈련 데이터에만 맞고 새 데이터에서 실패
- **과소적합:** 모델이 너무 단순 → 실제 패턴도 못 잡음

### LOS 11.c 요약
- **텍스트 분석:** 비정형 텍스트에서 투자 신호 추출 (단어 빈도, 감성 분석)
- **NLP:** 문맥을 이해하는 AI. 번역, 규제 준수 모니터링
- **위험 거버넌스:** 실시간 위험 평가, 스트레스 테스트
- **알고리즘 트레이딩:** 규칙 기반 자동 거래. HFT = 밀리초 단위 초고빈도 거래

**Q1.** 핀테크를 가장 정확하게 설명한 것은?

A. 기술을 금융 서비스 산업에 적용하는 것

**Q2.** 빅데이터 분석에 가장 유용한 기술 발전은?

A. 머신러닝
B. 높은 지연(high-latency) 수집
C. 사물인터넷 (IoT)

**풀이:**
- A. **정답** — 머신러닝은 대용량 데이터에서 패턴을 식별하고 모델링하는 데 유용
- B. 높은 지연은 데이터 분석에 불리 (실시간 처리 불가)
- C. IoT는 빅데이터의 소스(원천)이지, 분석 도구가 아님

**정답: A**
