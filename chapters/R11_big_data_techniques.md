## LOS 11.a: 핀테크 (FinTech)

### 왜 빅데이터와 AI가 투자에 중요한가?

당신이 주식을 사려고 합니다. 20년 전이라면 재무제표를 읽고 뉴스를 찾아보는 정도가 전부였습니다. 하지만 지금은?

어떤 헤지펀드는 **위성 사진**으로 월마트 주차장에 차가 몇 대 있는지 세서 매출을 예측합니다. 다른 펀드는 소셜 미디어에서 브랜드 감성을 실시간 분석합니다. 심지어 신용카드 거래 데이터로 스타벅스의 이번 주 결제 건수를 분기 실적 전에 추정하기도 합니다.

문제는 이 데이터의 양이 인간이 처리할 수 있는 범위를 넘어선다는 것입니다. 컴퓨터가 대신 패턴을 찾아주는 기술 — 머신러닝과 AI — 이 필요해진 것입니다. 빅데이터와 AI는 **남들이 못 보는 정보에서 먼저 패턴을 찾아 투자 우위를 확보하는 도구**입니다.

**핀테크(FinTech)** = 금융 서비스 산업에 적용되는 기술 발전. ATM에서 알고리즘 트레이딩까지 모두 포함됩니다.

## LOS 11.b: 빅데이터, 인공지능, 머신러닝

### 빅데이터의 3V

**Volume (규모):** 하루에 생성되는 데이터가 수십 페타바이트. 1페타바이트 = 신문 1,000억 장 분량.

**Velocity (속도):** 같은 정보라도 1초 먼저 아는 사람이 돈을 법니다. HFT 회사가 0.001초 만에 거래하는 이유.

**Variety (다양성):** 정형(스프레드시트), 반정형(HTML), 비정형(동영상, 소셜 미디어). 전체 데이터의 약 80%가 비정형.

| 데이터 소스 | 예시 | 왜 유용한가? |
|------------|------|------------|
| 개인 생성 | 트위터, 제품 리뷰, 검색 기록 | 소비자 심리 실시간 파악 |
| 기업 배기가스 | 은행 거래, POS 데이터 | 공식 발표 전 실적 추정 |
| IoT | GPS, 공장 센서, RFID | 실물 경제 실시간 추적 |

### 데이터 사이언스 5단계

빅데이터가 원유라면 데이터 사이언스는 정유 공장입니다.

1. **Capture** (수집) → 2. **Curation** (큐레이션: 불량·결측 처리) → 3. **Storage** (저장) → 4. **Search** (검색) → 5. **Transfer** (전송)

### 머신러닝

전통 통계는 "X와 Y는 직선 관계일 것이다"라고 가정합니다. 머신러닝은 "어떤 관계가 있는지 컴퓨터가 알아서 찾아라. 직선이든 곡선이든 상관없다."

**3단계 데이터 분할:** 시험 공부로 비유하면:
- **훈련 데이터** = 교과서
- **검증 데이터** = 모의시험
- **테스트 데이터** = 실제 시험

| 유형 | 특징 |
|------|------|
| **지도학습** | 정답 레이블 있음. "이 이메일은 스팸이다/아니다" |
| **비지도학습** | 정답 없이 패턴 발견. 고객 군집화 |
| **딥러닝** | 다층 신경망. 이미지·음성 인식 |

### 과적합과 과소적합

이것이 머신러닝에서 가장 중요한 개념입니다.

**과적합 (Overfitting):** 교과서 문제는 만점인데 실전에서 0점. 모델이 훈련 데이터의 노이즈까지 외워버린 상태. "월요일에 비가 오면 주가가 올랐다"는 우연의 패턴을 법칙으로 학습.

**과소적합 (Underfitting):** 교과서도 제대로 안 읽은 상태. 모델이 너무 단순해서 실제 패턴도 못 잡음.

| 문제 | 증상 |
|------|------|
| 과적합 | 훈련 99%, 테스트 50% |
| 과소적합 | 훈련 60%, 테스트 55% |

**블랙박스 문제:** 딥러닝이 "이 주식을 사라"고 했는데 **왜** 그런지 사람이 이해할 수 없습니다. 규제 환경에서 "AI가 그렇게 판단했습니다"라고 답할 수 없습니다.

## LOS 11.c: 투자 관리에서의 활용

### 텍스트 분석

비정형 텍스트에서 투자 신호 추출. ECB 총재 발언의 톤을 밀리초 만에 분석하여 유로화 매매.

### 자연어 처리 (NLP)

텍스트 분석이 "단어를 세는 것"이라면, NLP는 "문맥을 이해하는 것". "이 회사 실적이 죽인다"에서 "죽인다"가 긍정인지 부정인지 판단.

### 알고리즘 트레이딩

1,000만 주를 한 번에 사면 가격이 폭등합니다. 알고리즘이 하루 동안 100번에 나눠서, 거래량이 많은 시간에 더 많이 삽니다. **HFT:** 밀리초 단위 거래. 미국 주식 거래량의 약 50%.
